# Design References

Use this list to keep the visual direction for `ui` consistent across
apps. Add sites when they represent a real preference for the component library:
clean typography, dense but readable controls, restrained colour, strong
developer-tool ergonomics.

This is an influence list, not a cloning brief. Record what to learn from each
site and what not to copy.

| Site | What to learn from it | Avoid copying |
|---|---|---|
| [Bun](https://bun.com/) | Clean developer-tool UI, dark-first surfaces, system sans body text, JetBrains Mono-style technical text, compact examples, restrained borders. | Bun branding, marketing sections, mascot/product-specific illustration, benchmark layout as an app pattern. |

## Adding References

When adding a site, include:

- The public URL.
- The specific traits worth borrowing.
- The traits to avoid so the component library stays `ui`, not a replica.
- A date if the site is likely to change materially.
